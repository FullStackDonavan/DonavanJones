## Who This Is For

You're building an AI-backed API and you don't want to spend the first week re-solving authentication, vector search, and Docker setup before you can even start on the feature that actually matters. This boilerplate is the backend foundation distilled from the [Backend Engineering series](/categories/backend-engineering) — the scaffolding already done, so you start on day one with the actual product logic.

## What's Inside

- **FastAPI starter** — a project structure that scales past a single `main.py` file
- **Authentication** — already implemented, not a placeholder comment
- **Vector search** — wired up for semantic queries against your own data
- **Embedding services** — turning raw content into vectors as part of the normal request flow
- **Docker files** — for local development and for production builds
- **CI/CD examples** — so tests and deploys run automatically instead of by hand

## Why It's Different From Just Reading the Blog

The blog series covers the architecture decisions and the reasoning behind them. This boilerplate is the repo you clone and build on top of — the auth, the vector search, and the deployment pipeline already working, instead of being rebuilt from a blog post.
